# Чат-бот с ML-классификатором интентов и FSM

Консольный чат-бот с гибридной архитектурой, объединяющей машинное обучение для распознавания намерений, конечный автомат для управления диалогом, интеграцию с API погоды и хранение данных в SQLite.

## Описание

Проект представляет собой интеллектуального консольного чат-бота, построенного на гибридной архитектуре. Основной инновацией является использование **ML-классификатора интентов на основе Random Forest**, обученного на размеченном датасете, что позволяет боту понимать намерения пользователя без жестких регулярных выражений. При этом сохранена обратная совместимость с паттернами для fallback-обработки.

**Архитектура классификации интентов**:
- Модель обучается на датасете `intent_dataset_final.csv` с помощью скрипта `SKLearn.py`
- Векторизация текста выполняется через **TF-IDF** с настройками: 5000 признаков, использование униграмм и биграмм
- Классификатор — **Random Forest** (200 деревьев, max_depth=20, class_weight='balanced')
- Предобработка текста использует **spaCy** с моделью `ru_core_news_sm` (лемматизация, удаление стоп-слов и пунктуации) с fallback на regex-очистку
- Классификатор загружается в боте через синглтон `IntentClassifier` и возвращает интент с уровнем уверенности (порог 0.6)

**Управление диалогом через FSM (конечный автомат)**:
- Реализован в классе `DialogManager` с состояниями `START` и `WAIT_CITY`
- Состояния сохраняются в базе данных (поле `state` в таблице `chat_log`), что позволяет восстанавливать контекст после перезапуска
- При запросе погоды без указания города бот переходит в состояние ожидания города, уточняет его, после чего возвращает прогноз
- Команда «отмена» сбрасывает текущее состояние

**Интеграция компонентов**:
- `bot_core.py` — основной класс `ChatBot`, который:
  - Загружает ML-классификатор при инициализации
  - Получает текущее состояние пользователя из БД
  - Сначала пытается обработать сообщение через FSM (`dialog_manager`)
  - Затем использует ML-классификатор для определения интента (greeting, farewell, weather, set_name, help, stats, schedule, yes_no, unknown)
  - При низкой уверенности (<0.6) или отсутствии модели использует fallback на регулярные выражения из `patterns.py`
  - Логирует каждое сообщение с привязкой к пользователю и состоянию FSM

**Модульная структура проекта**:
| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа |
| `bot_core.py` | Основной класс бота, интеграция ML и FSM |
| `dialog_manager.py` | FSM для управления многошаговыми диалогами |
| `intent_classifier.py` | Загрузка ML-модели, предсказание интентов |
| `SKLearn.py` | Обучение Random Forest на датасете |
| `patterns.py` | Регулярные выражения и обработчики (fallback) |
| `database.py` | SQLite: пользователи, логи, погодные запросы, состояния |
| `weather_api.py` | Интеграция с Weatherstack API |
| `test.py` | Утилита тестирования извлечения городов через spaCy |

**Особенности реализации**:
- Гибридный подход: ML для основных интентов, regex как резервный механизм
- Персистентность FSM: состояния сохраняются в БД и восстанавливаются между сообщениями
- Идентификация пользователей по имени с ведением персональной статистики
- Поддержка запросов погоды с указанием дат (сегодня, завтра, дни недели) и валидацией на 7 дней вперед
- Логирование всех взаимодействий с привязкой к пользователю и состоянию FSM для полной воспроизводимости
