# Классификатор интентов на spaCy word vectors и логистической регрессии

Модель машинного обучения для классификации текстовых запросов по интентам с использованием векторных представлений слов (word embeddings) из spaCy и логистической регрессии.

## Описание

Проект представляет собой легковесный классификатор интентов для русского языка, построенный на предобученных векторных представлениях слов из библиотеки spaCy. В отличие от TF-IDF, этот подход позволяет улавливать семантическую близость между словами, что повышает качество классификации на ограниченных датасетах.

**Архитектура решения**:
- **Векторизация текста** выполняется с помощью spaCy и модели `ru_core_news_md` (средняя модель для русского языка). Каждый текст преобразуется в плотный вектор (300-мерный) через `doc.vector` — усреднение векторов всех токенов предложения. Такой подход сохраняет семантический смысл и не требует ручной настройки признаков
- **Классификатор** — логистическая регрессия с гиперпараметрами: максимальное количество итераций 2000, балансировка весов классов (`class_weight='balanced'`), параметр регуляризации C=1.0. Логистическая регрессия выбрана за простоту, интерпретируемость и эффективность на задачах с небольшим количеством признаков
- **Обучение** происходит на датасете `dataset.csv` с колонками `text` и `intent`. Модель обучается на всех доступных данных (без разделения на обучающую/тестовую выборки для демонстрационных целей)

**Структура проекта**:
| Файл | Назначение |
|------|------------|
| `main.py` | Основной скрипт: загрузка модели spaCy, чтение датасета, векторизация, обучение логистической регрессии, сохранение модели |
| `dataset.csv` | Размеченный датасет с текстами и интентами |
| `random_forest_model/classifier.pkl` | Сохраненная модель классификатора (pickle/joblib) |

**Особенности реализации**:
- Используется модель spaCy `ru_core_news_md` (300-мерные векторы), которая предобучена на больших русскоязычных корпусах и учитывает семантику слов
- Векторизация выполняется усреднением векторов всех слов в тексте, что позволяет обрабатывать тексты произвольной длины
- Логистическая регрессия обучена с балансировкой весов классов для работы с несбалансированными данными
- После обучения модель сохраняется в формате joblib для последующего использования в чат-боте
- Предусмотрена тестовая проверка на примере фразы для быстрой верификации работы модели

**Преимущества подхода**:
- Семантическое понимание: близкие по смыслу слова (например, «осадки» и «дождь») дают схожие векторы
- Простота реализации и быстрая векторизация без сложной предобработки
- Компактность модели: логистическая регрессия требует мало памяти и быстро выполняет предсказания
- Возможность легкого расширения датасета и переобучения
