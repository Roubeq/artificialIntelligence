# Голосовой чат-бот с RuBERT, FSM, навыками и синтезом речи (Coqui TTS)

Консольный голосовой чат-бот с расширенной архитектурой, объединяющей **RuBERT** для классификации интентов, **конечный автомат (FSM)** для управления диалогом, **систему навыков (Skills)** для модульной обработки запросов, интеграцию с API погоды, хранение данных в SQLite и **синтез речи (Coqui TTS)** для голосового вывода ответов.

## Описание

Проект представляет собой полнофункционального голосового чат-бота с современной модульной архитектурой. Ключевые инновации — использование **RuBERT (DeepPavlov/rubert-base-cased)** для высокоточного распознавания намерений, **система навыков**, позволяющая гибко расширять функциональность, и **синтез речи Coqui TTS** для озвучивания ответов с поддержкой кэширования и асинхронного воспроизведения.

**Архитектура классификации интентов**:
- **Модель**: RuBERT от DeepPavlov, дообученная на датасете с 13+ классами (greeting, farewell, weather, weather_city_only, set_name, schedule, yes_no, stats, help, unknown, time, date, smalltalk, follow_up)
- **Токенизация**: BERT-токенизатор с максимальной длиной последовательности 32 токена
- **Обучение**: 3 эпохи, batch size 2, сохранение модели с маппингом меток
- **Порог уверенности**: 0.5, при низкой уверенности используется fallback на регулярные выражения

**Синтез речи (Coqui TTS)**:
- **Модель**: YourTTS (`tts_models/multilingual/multi-dataset/your_tts`) с поддержкой русского языка
- **Кэширование**: WAV-файлы кэшируются на диске и в памяти для быстрого повторного воспроизведения
- **Асинхронное воспроизведение**: синтез и проигрывание выполняются в отдельном потоке без блокировки основного диалога
- **Fallback**: при отсутствии референсного голоса используется английская модель с транслитерацией кириллицы
- **Нормализация текста**: удаление эмодзи, замена единиц измерения («км/ч» → «километров в час»), очистка от лишних пробелов
- **Переменные окружения**:
  - `COQUI_TTS=0` — отключение синтеза речи
  - `COQUI_TTS_MODEL` — выбор модели TTS
  - `COQUI_TTS_SPEAKER_WAV` — путь к референсному WAV-файлу для клонирования голоса
  - `COQUI_TTS_LANGUAGE` — язык (по умолчанию `ru`)

**Архитектура навыков (Skills)**:
- Модуль `router.py` отвечает за маршрутизацию интентов к соответствующим обработчикам
- Функция `resolve_effective_intent()` уточняет интент с учетом контекста диалога
- Функция `route_intent()` вызывает конкретный обработчик для каждого типа интента
- Контекст диалога (последний интент, город, реплики) хранится в `DialogManager`

**Управление диалогом через FSM**:
- Конечный автомат с состояниями `START` и `WAIT_CITY`
- Состояния сохраняются в SQLite для восстановления контекста
- Поддержка follow-up: после запроса погоды пользователь может уточнить «а в Москве?»

**Структура проекта**:
| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа |
| `bot_core.py` | Основной класс бота, интеграция RuBERT, FSM, Skills, TTS |
| `coqui_tts.py` | Синтез речи: загрузка модели, кэширование, асинхронное воспроизведение |
| `intent_classifier_bert.py` | Загрузка и инференс RuBERT-модели |
| `train_intent_model.py` | Дообучение RuBERT на датасете |
| `dialog_manager.py` | FSM + хранение контекта (последний интент, город, реплики) |
| `patterns.py` | Регулярные выражения (fallback), извлечение городов и дат через spaCy |
| `database.py` | SQLite: пользователи, логи с состояниями, статистика погоды |
| `weather_api.py` | Интеграция с Weatherstack API |
| `dataset.csv` | Размеченный датасет с интентами |

**Особенности реализации**:
- **Голосовой вывод**: все ответы бота озвучиваются через Coqui TTS с возможностью отключения
- **Кэширование аудио**: повторяющиеся фразы синтезируются один раз и берутся из кэша
- **Асинхронность**: воспроизведение не блокирует ввод следующего сообщения
- **Транслитерация**: для английской fallback-модели кириллица автоматически транслитерируется
- **Follow-up контекст**: бот понимает уточняющие вопросы благодаря сохранению контекста
- **Smalltalk**: адекватные ответы на «спасибо», «окей», «круто»
