# Консольный чат-бот с поддержкой диалога и NLP

Консольный чат-бот с использованием spaCy для обработки естественного языка, поддержкой диалоговых состояний, интеграцией с API погоды и хранением данных в SQLite.

## Описание

Проект представляет собой консольного чат-бота с расширенными возможностями обработки естественного языка. В отличие от простых regex-паттернов, бот использует библиотеку spaCy для распознавания именованных сущностей (местоположений, дат) и лемматизации, что позволяет более гибко обрабатывать запросы пользователей.

Ключевым нововведением является **диалоговый менеджер** (модуль `dialog_manager.py`), который реализует конечный автомат с состояниями. Это позволяет боту вести многошаговые диалоги: например, если пользователь спрашивает «Какая погода?» без указания города, бот переходит в состояние `WAIT_CITY` и запрашивает город, после чего обрабатывает полученный ответ. Состояния сохраняются в базе данных, что позволяет восстанавливать контекст диалога между сообщениями.

**Распознавание городов и дат** осуществляется через spaCy:
- Сущности с метками `LOC` (локация) и `GPE` (геополитическая единица) извлекаются из текста и используются для запроса погоды
- Даты распознаются на уровне лемм: «сегодня», «завтра», «послезавтра», а также дни недели («в понедельник», «в среду»)
- Прогноз погоды доступен на 7 дней вперед с валидацией запрошенной даты

**Архитектура базы данных** расширена полем `state` в таблице `chat_log`, что позволяет отслеживать состояние диалога в момент каждого сообщения. Это обеспечивает полную воспроизводимость истории взаимодействия.

**Модульная структура** проекта:
- `main.py` — точка входа
- `bot_core.py` — основной цикл обработки сообщений, интеграция с диалоговым менеджером
- `dialog_manager.py` — управление состояниями диалога, обработка многошаговых сценариев
- `patterns.py` — регулярные выражения и обработчики команд с поддержкой spaCy
- `database.py` — работа с SQLite: инициализация, сохранение пользователей, логирование с состоянием
- `weather_api.py` — интеграция с Weatherstack API
- `test.py` — утилита для тестирования извлечения городов через spaCy

**Особенности реализации**:
- Пользователь идентифицируется по имени, после представления все сообщения привязываются к его ID
- Ведется персональная статистика: количество сообщений, дата первого/последнего визита, количество запросов погоды
- Команда «отмена» или «выход» позволяет прервать текущий диалог и вернуться в начальное состояние
- Логирование в базу данных включает не только сообщения, но и состояние диалога на момент обработки

Такой подход позволяет боту обрабатывать естественные формулировки запросов (например, «Какая погода?» → уточнение города → «Санкт-Петербург» → ответ) и расширять функциональность добавлением новых диалоговых сценариев без изменения ядра.
