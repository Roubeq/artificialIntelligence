# Интеллектуальный чат-бот с RuBERT и FSM

Консольный чат-бот с гибридной архитектурой, использующий **RuBERT (DeepPavlov/rubert-base-cased)** для классификации интентов, конечный автомат для управления диалогом, интеграцию с API погоды и хранение данных в SQLite.

## Описание

Проект представляет собой интеллектуального чат-бота, ключевой особенностью которого является использование **предобученной русскоязычной BERT-модели** для точного распознавания намерений пользователя. RuBERT обеспечивает высокое качество классификации благодаря глубокому пониманию контекста и семантики русскоязычных текстов.

**Архитектура классификации интентов**:
- **Модель**: RuBERT от DeepPavlov (`DeepPavlov/rubert-base-cased`), дообученная на размеченном датасете
- **Интенты**: 10 классов — `greeting`, `farewell`, `weather`, `weather_city_only`, `set_name`, `schedule`, `yes_no`, `stats`, `help`, `unknown`
- **Векторизация**: встроенный токенизатор BERT с максимальной длиной последовательности 32 токена
- **Обучение**: 3 эпохи, batch size 2, оптимизация через Hugging Face Trainer
- **Порог уверенности**: предсказание принимается при confidence ≥ 0.5, иначе используется fallback

**Управление диалогом через FSM**:
- Конечный автомат с состояниями `START` и `WAIT_CITY` в классе `DialogManager`
- При запросе погоды без указания города бот переходит в состояние ожидания, уточняет город, затем возвращает прогноз
- Состояния сохраняются в SQLite (поле `state` в таблице `chat_log`) для восстановления контекста после перезапуска
- Команда «отмена» сбрасывает текущее состояние

**Гибридная обработка сообщений**:
1. FSM обрабатывает многошаговые диалоги
2. RuBERT классифицирует интент с оценкой уверенности
3. При низкой уверенности (<0.5) используется fallback на регулярные выражения из `patterns.py`

**Структура проекта**:
| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа |
| `bot_core.py` | Основной класс бота, интеграция RuBERT, FSM и fallback-паттернов |
| `intent_classifier_bert.py` | Загрузка и инференс RuBERT-модели |
| `train_intent_model.py` | Скрипт дообучения RuBERT на датасете |
| `test_model.py` | Тестирование обученной модели |
| `dialog_manager.py` | FSM для многошаговых диалогов |
| `patterns.py` | Регулярные выражения (fallback), извлечение городов и дат через spaCy |
| `database.py` | SQLite: пользователи, логи с состояниями FSM, статистика погоды |
| `weather_api.py` | Интеграция с Weatherstack API |
| `expand_dataset.py` | Скрипт расширения датасета новыми примерами |
| `dataset.csv` | Размеченный датасет для обучения RuBERT |
| `intent_model/` | Директория с сохраненной моделью и токенизатором |

**Особенности реализации**:
- **RuBERT**: использование современной трансформерной архитектуры для русского языка
- **Персистентность FSM**: состояние диалога сохраняется в БД
- **Идентификация пользователей**: сохранение имени с ведением персональной статистики
- **Обработка дат**: поддержка «сегодня», «завтра», дней недели, валидация на 7 дней
- **Расширяемый датасет**: возможность добавления новых примеров через `expand_dataset.py`
- **Логирование**: каждое сообщение в БД с привязкой к пользователю и состоянию FSM
