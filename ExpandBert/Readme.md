# Интеллектуальный чат-бот с RuBERT, FSM и архитектурой навыков (Skills)

Консольный чат-бот с расширенной архитектурой, объединяющей **RuBERT** для классификации интентов, **конечный автомат (FSM)** для управления диалогом, **систему навыков (Skills)** для модульной обработки запросов, интеграцию с API погоды и хранение данных в SQLite.

## Описание

Проект представляет собой полнофункционального чат-бота с современной модульной архитектурой. Ключевые инновации — использование **RuBERT (DeepPavlov/rubert-base-cased)** для высокоточного распознавания намерений и **система навыков**, позволяющая гибко расширять функциональность без изменения ядра.

**Архитектура классификации интентов**:
- **Модель**: RuBERT от DeepPavlov, дообученная на датасете с 13+ классами (greeting, farewell, weather, weather_city_only, set_name, schedule, yes_no, stats, help, unknown, time, date, smalltalk, follow_up)
- **Токенизация**: BERT-токенизатор с максимальной длиной последовательности 32 токена
- **Обучение**: 3 эпохи, batch size 2, сохранение модели с маппингом меток
- **Порог уверенности**: 0.5, при низкой уверенности используется fallback на регулярные выражения

**Архитектура навыков (Skills)**:
- Модуль `router.py` (не показан в файлах, но импортируется) отвечает за маршрутизацию интентов к соответствующим обработчикам
- Функция `resolve_effective_intent()` уточняет интент с учетом контекста диалога (например, превращает `weather_city_only` в полноценный запрос погоды)
- Функция `route_intent()` вызывает конкретный обработчик для каждого типа интента
- Контекст диалога (последний интент, город, реплики) хранится в `DialogManager` и используется для follow-up вопросов

**Управление диалогом через FSM**:
- Конечный автомат с состояниями `START` и `WAIT_CITY`
- Состояния сохраняются в SQLite для восстановления контекста
- Поддержка follow-up: после запроса погоды пользователь может уточнить «а в Москве?»
- Команда «отмена» сбрасывает текущее состояние

**Структура проекта**:
| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа |
| `bot_core.py` | Основной класс бота, интеграция RuBERT, FSM, Skills и fallback |
| `intent_classifier_bert.py` | Загрузка и инференс RuBERT-модели |
| `train_intent_model.py` | Дообучение RuBERT на датасете |
| `dialog_manager.py` | FSM + хранение контекста (последний интент, город, реплики) |
| `patterns.py` | Регулярные выражения (fallback), извлечение городов и дат через spaCy |
| `database.py` | SQLite: пользователи, логи с состояниями, статистика погоды |
| `weather_api.py` | Интеграция с Weatherstack API |
| `dataset.csv` | Размеченный датасет с интентами (date, time, smalltalk, follow_up и др.) |

**Интенты модели**:
- `greeting`, `farewell`, `weather`, `weather_city_only`, `set_name`, `schedule`, `yes_no`, `stats`, `help`, `unknown`
- `date` — запросы даты и дня недели
- `time` — запросы текущего времени
- `smalltalk` — общие фразы (спасибо, окей, круто, как дела)
- `follow_up` — уточняющие вопросы («а в питере?», «и что дальше?», «продолжай»)

**Особенности реализации**:
- **Follow-up контекст**: после ответа о погоде в Москве пользователь может спросить «а в Питере?», и бот поймет, что речь о погоде
- **Модульные навыки**: каждый интент обрабатывается отдельной функцией, новые навыки добавляются через роутер
- **Запоминание города**: последний запрошенный город сохраняется в контексте
- **Поддержка дат и времени**: отдельные интенты для запросов даты и времени
- **Smalltalk**: бот адекватно реагирует на «спасибо», «окей», «круто»
