# Голосовой чат-бот с RuBERT, FSM, навыками, Whisper ASR и Coqui TTS

Полноценный голосовой ассистент с распознаванием речи (Whisper), классификацией интентов (RuBERT), управлением диалогом (FSM), модульными навыками, синтезом речи (Coqui TTS) и хранением данных (SQLite).

## Описание

Проект представляет собой законченное решение для голосового взаимодействия с ИИ-ассистентом. Пользователь может говорить в микрофон, бот распознаёт речь через **Whisper**, понимает намерение через **RuBERT**, обрабатывает запрос через **систему навыков** и отвечает голосом через **Coqui TTS**.

**Архитектура обработки речи**:
- **ASR (распознавание речи)**: Используется модель **Whisper (`base`)** от OpenAI. Запись длительностью 4 секунды с частотой 16 кГц сохраняется в WAV и транскрибируется в текст с очисткой от спецсимволов.
- **NLU (понимание естественного языка)**: **RuBERT (DeepPavlov/rubert-base-cased)** классифицирует интенты (greeting, farewell, weather, schedule, yes_no, stats, help, time, date, smalltalk, unknown и др.) с порогом уверенности 0.5.
- **Диалоговый менеджер**: FSM с состояниями `START` и `WAIT_CITY` для сбора недостающих слотов (город для погоды). Контекст сохраняется в SQLite.
- **Навыки (Skills)**: Модульная архитектура через `router.py` + `resolve_effective_intent()` поддерживает follow-up («а в Москве?») и переиспользование логики.
- **TTS (синтез речи)**: **Coqui TTS (YourTTS)** с русской речью, кэшированием WAV-файлов на диске и в памяти, асинхронным воспроизведением (без блокировки ввода). Поддерживается клонирование голоса через `COQUI_TTS_SPEAKER_WAV`.

**Полный цикл работы**:
1. Пользователь говорит в микрофон (4 секунды записи)
2. Whisper транскрибирует речь → текст
3. RuBERT определяет интент
4. Если интент `weather_city_only` — FSM переходит в `WAIT_CITY` и запрашивает город
5. Соответствующий навык (погода, статистика, дата/время, yes/no) генерирует ответ
6. Coqui TTS синтезирует ответ (асинхронно, с кэшированием)
7. Бот выводит текст в консоль и проигрывает голос

**Структура проекта**:
| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа |
| `bot_core.py` | Основной класс бота: интеграция всех компонентов, цикл обработки |
| `voice_module.py` | Запись с микрофона, распознавание через Whisper, очистка текста |
| `coqui_tts.py` | Синтез речи: загрузка модели, нормализация, кэширование, асинхронное воспроизведение |
| `intent_classifier_bert.py` | Загрузка и инференс RuBERT-модели |
| `train_intent_model.py` | Дообучение RuBERT на датасете |
| `dialog_manager.py` | FSM + хранение контекста (последний интент, город, реплики) |
| `patterns.py` | Fallback-обработка (регулярные выражения), извлечение городов и дат через spaCy |
| `database.py` | SQLite: пользователи, логи с состояниями, статистика погоды |
| `weather_api.py` | Интеграция с Weatherstack API |
| `skills/router.py` | Маршрутизация интентов к обработчикам и follow-up логика |
| `dataset.csv` | Размеченный датасет для обучения RuBERT |

**Интенты модели**:
- `greeting`, `farewell`, `weather`, `weather_city_only`, `set_name`, `schedule`, `yes_no`, `stats`, `help`, `unknown`
- `date` — запросы даты и дня недели
- `time` — запросы текущего времени
- `smalltalk` — общие фразы (спасибо, окей, круто, как дела)
- `follow_up` — уточняющие вопросы («а в питере?», «и что дальше?»)

**Переменные окружения**:
| Переменная | Назначение | По умолчанию |
|------------|------------|---------------|
| `COQUI_TTS` | Включение/отключение TTS | `1` (вкл) |
| `COQUI_TTS_MODEL` | Модель синтеза речи | `tts_models/multilingual/multi-dataset/your_tts` |
| `COQUI_TTS_SPEAKER_WAV` | Референсный WAV для клонирования голоса | `None` |
| `COQUI_TTS_LANGUAGE` | Язык TTS | `ru` |
