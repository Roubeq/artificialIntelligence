# Интеллектуальный чат-бот с классификатором на spaCy word vectors

Консольный чат-бот с гибридной архитектурой, использующий семантические векторные представления слов (word embeddings) из spaCy для классификации интентов, конечный автомат для управления диалогом и интеграцию с API погоды.

## Описание

Проект представляет собой интеллектуального чат-бота, ключевой особенностью которого является **классификатор интентов на основе векторных представлений слов**. В отличие от TF-IDF или регулярных выражений, этот подход использует плотные семантические векторы (word embeddings), что позволяет боту понимать смысловую близость между словами и лучше обрабатывать естественные формулировки.

**Архитектура классификации интентов**:
- **Векторизация текста** выполняется с помощью spaCy и модели `ru_core_news_md` (средняя русскоязычная модель с 300-мерными векторами). Каждое сообщение преобразуется в вектор через `doc.vector` — усреднение векторов всех слов предложения
- **Классификатор** — логистическая регрессия, обученная на размеченном датасете. Модель сохраняется в формате joblib и загружается при запуске бота
- **Порог уверенности** — предсказание принимается только при уверенности ≥ 0.6, иначе бот переключается на fallback-обработку (регулярные выражения)

**Управление диалогом через FSM**:
- Реализован конечный автомат с состояниями `START` и `WAIT_CITY` в классе `DialogManager`
- При запросе погоды без указания города бот переходит в состояние ожидания, уточняет город, после чего возвращает прогноз
- Состояния сохраняются в базе данных SQLite (поле `state` в таблице `chat_log`), что позволяет восстанавливать контекст между сообщениями и после перезапуска
- Команда «отмена» сбрасывает текущее состояние и возвращает в `START`

**Гибридная обработка сообщений**:
1. FSM обрабатывает многошаговые диалоги (ожидание города)
2. ML-классификатор определяет интент (greeting, farewell, weather, set_name, help, stats, schedule, yes_no, unknown)
3. При низкой уверенности (<0.6) или отсутствии модели используется fallback на регулярные выражения из `patterns.py`

**Структура проекта**:
| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа |
| `bot_core.py` | Основной класс бота, интеграция ML, FSM и fallback-паттернов |
| `intent_classifier.py` | Загрузка модели (spaCy + LogReg), предсказание интентов с векторизацией через `doc.vector` |
| `dialog_manager.py` | FSM для управления многошаговыми диалогами |
| `patterns.py` | Регулярные выражения и обработчики (fallback), извлечение городов и дат через spaCy |
| `database.py` | SQLite: пользователи, логи с состояниями FSM, статистика запросов погоды |
| `weather_api.py` | Интеграция с Weatherstack API |
| `test.py` | Утилита тестирования извлечения городов через spaCy |

**Особенности реализации**:
- **Семантическая векторизация**: использование предобученной модели `ru_core_news_md` (300-мерные векторы) позволяет улавливать смысловые связи («осадки» → «дождь», «холодно» → «температура»)
- **Персистентность FSM**: состояние диалога сохраняется в БД, что обеспечивает устойчивость к перезапускам
- **Идентификация пользователей**: сохранение имени в таблице `users` с ведением персональной статистики (количество сообщений, запросов погоды)
- **Логирование**: каждое сообщение записывается в БД с привязкой к пользователю и состоянию FSM

**Преимущества подхода**:
- Семантическое понимание благодаря word embeddings
- Компактность и быстрота логистической регрессии
- Надежный fallback через регулярные выражения
- Сохраняемость контекста диалога через FSM в БД

Данная архитектура позволяет боту гибко обрабатывать естественные запросы, вести контекстные диалоги и легко расширяться добавлением новых интентов через дообучение классификатора.
